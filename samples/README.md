# 样本文档清单

本目录用于登记两周轻量实验的 10 份真实样本文档。

当前状态：尚未提供真实 PDF/Word/Markdown 文件。

请将真实文件放入：

```text
.\samples\raw\
```

并补齐下表。

也可以直接填写结构化清单：

```text
.\samples\sample-manifest.csv
```

如果真实文档还没有最终入选，先让文档 owner、领域确认人或任务评分人填候选收集表：

```text
.\samples\document-intake-template.csv
```

填表示例：

```text
.\samples\document-intake-example.csv
```

owner 请求和反馈状态跟踪：

```text
.\samples\owner-response-tracker.csv
```

候选表用于收集来源路径、版本、owner、密级、是否允许进入实验等信息；入选后再复制文件到 `samples/raw/` 并写入正式 manifest。

`source_location` 建议填写可访问的本地盘符路径或共享盘路径，脚本会检查路径是否存在。若填写 HTTP/HTTPS 链接，需要 owner 额外确认该文档可下载、可进入实验环境。

文档放入 `samples/raw/` 后，可先生成 manifest 草稿：

```powershell
powershell -ExecutionPolicy Bypass -File ".\scripts\import-sample-docs.ps1" -WriteDraft
```

确认草稿无误后再应用到正式清单：

```powershell
powershell -ExecutionPolicy Bypass -File ".\scripts\import-sample-docs.ps1" -Apply
```

## 样本选择要求

10 份文档需要尽量覆盖：

- 普通文本 PDF。
- 表格密集 PDF。
- 多栏 PDF。
- 至少 1 份扫描件或 OCR 风险文档。
- 至少 1 份供应商文档。
- 至少 1 份内部 SPEC。
- 至少 1 份内部设计文档。
- 至少 1 份测试或问题复盘文档。

如果找不到某类文档，需要在状态列说明原因。
